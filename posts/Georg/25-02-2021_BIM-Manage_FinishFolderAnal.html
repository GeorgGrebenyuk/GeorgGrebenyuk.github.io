Верификация файлов участниками в своих Итоговых контрольных точках

Дата публикации: 25-26.02.2021г.

Курс BIM-менеджмент близится к завершению, дата последней контрольной точки установлена на 9 марта. В связи с этим, необходимо было изменить логику проверок прошлых анализаторов данных, отталкиваясь от структуры файлов, приведенной в ТЗ. Примерно за неделю до этого, в конце прошлой недели Александ Евгеньевич поппросил сделать подобное приложение для самих участников курса - чтобы они запускали его на своей стороне и шла проверка имен файлов, а также наличия нужных файлов по папкам в рамках данной "Итоговой".

Я решил сперва реализовать приложение для проверки файлов на стороне участников, а потом, собрав негативный feedback сделать уже для извлечения данных нанашей стороне. На этот раз, в отличие от приватного репозитория AppsForVC, я решил разместить это приложение в публичном репозитории AppsForVC-NN_Open.

В рамках данной задачи я приоткрыл для себя элемент DataGridView в WinForms, а также освоил хоть в базовом варианте язык разметки markdown для оформления репозитория.

Структура и логика приложения

Структура ничем не отличается от устоявшейся схемы - класс Form1 с элементами управления интерфесом, public static class Actions для выполнения непосредственно логики проверок и public static class Technical для промежуточных операций как-то проверка корректности имени файла, проверка Revit-файлов на наличие временных и т.д. (собственно говоря - это 2 функции тут только).

Процесс работы выглядит следующим образом:

  1. Пользователь запускает программу и выбирает каталог "Исходящие" в рамках своего участка. Это нужно во-первых, чтобы получить наименование участка взяв имя родительской категории у данной (и в дальнейшем использовать спарку "квадрат"-"номер участка"), а во вторых - чтобы проверить корректность именования папки контрольной итоговой точки
